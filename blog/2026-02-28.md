# Week of February 28, 2026

## Testing the foundations before we touch them

This week was about slowing down to speed up. We're preparing for a series of changes to core systems — subscriptions, referrals, reputation — and before we change anything, we need to know that what's already working doesn't break.

### Integration tests for critical paths

We built an integration testing layer from scratch. Not unit tests that mock everything and prove nothing. Real tests that hit the database, create actual users, publish actual posts, and verify the full chain works end to end.

The first batch covers the paths that matter most:

- **Post creation** — a user creates a post, it lands in the database with the right structure, the right visibility, the right associations.
- **Home feed and user profile feed** — posts show up where they should, in the order they should, with the right filters applied.
- **Post interactions** — likes, replies, and the notifications they trigger. When someone replies to your post, does the notification actually get created? We now have a definitive answer.
- **User profiles** — profile data, follower counts, post counts, all reflecting reality.
- **Moderation flow** — the community voting pipeline, from report to resolution.

Why this matters: we're about to modify how subscriptions and referrals work. These tests give us a safety net. We run them before and after every change, and if something breaks, we know immediately — not three days later when a user reports it.

### Research: trials, referrals, and reputation scoring

The other half of the week was pure investigation. No code written, no features shipped. Just reading, tracing, and documenting.

**Trials and referrals** — We traced every line of code involved in the trial subscription flow and the referral reward chain. The goal was to understand the current state before planning changes. What we found was that these two systems are deeply entangled in ways that create problems. Trial logic gates referral rewards. Referral rewards assume trials exist. Disabling one breaks the other. We documented seven distinct issues across the chain and filed them. Now we have a clear map of what needs to be decoupled before either system can evolve independently.

**Reputation scoring** — We audited the profile reputation system against its design spec. The system uses five pillars (Quality, Wellbeing, Altruism, Respect, Competence) to produce a score out of 100. We verified which pillars have real calculation logic behind them and which are stubs returning zero. Three of five are live with actual formulas. Two are placeholders. We also found a scoring asymmetry in the reaction weights that needs a decision — one of those things where the code drifted from the original intent and nobody noticed because the numbers still looked plausible.

### What's next

We now have a tested foundation and a documented map of the systems we're about to change. Next week we start the actual work — decoupling trials from referrals so each can function on its own terms.
